class wte.views.user.PasswordValidator(*args, **kw)[source]¶
The PasswordValidator handles the checking of

user-provided passwords against the database to allow / dissallow login.

Login is disallowed, if the password does not match the e-mail address or if the User‘s validation_token is still set, meaning they have not confirmed their registration.

Requires a SQLAlchemy database session to be available via state.dbsession.

class wte.views.user.UniqueEmailValidator(*args, **kw)[source]¶
The UniqueEmailValidator checks that the given

e-mail address is not already used by a User.

Requires a SQLAlchemy database session to be available via state.dbsession. If a state.userid is provided, then the User with that id can have the same e-mail address.

wte.views.user.action(request)[source]¶

Handles the /users/action URL, applying the given action to the list of selected users. Requires that the current User has the “admin.users.view” Permission.

wte.views.user.confirm(request)[source]¶

Handles the “/users/{uid}/confirm/{token}” URL, validating that the user with the given {uid} received the {token} at the given e-mail address.

If confirmed, will send an e-mail with a new, random password.

wte.views.user.create_user_crumbs(request, crumbs)[source]¶

Creates the base-list of breadcrumbs, depending on the current users authorisation level.

wte.views.user.delete(request)[source]¶

Handles the “/users/{uid}/delete” URL, providing the form and backend functionality for deleting a User. Also deletes all the data that is linked to that User.

wte.views.user.edit(request)[source]¶

Handles the “/users/{uid}/edit” URL, providing the form and backend functionality to update the user’s profile.

wte.views.user.forgotten_password(request)[source]¶

Handles the “/users/forgotten-password” URL, showing the form where the user can provide their e-mail address. If the User has a validation_token, then an e-mail with a new validation token is sent, otherwise an e-mail with a new, random password is sent.

wte.views.user.init(config)[source]¶

Adds the user-specific routes (route name, URL pattern, handler):

  • users – /users – users()
  • users.action – /users/action – action()
  • user.login – /users/login – login()
  • user.logout – /users/logout – logout()
  • user.register – /users/register – register()
  • user.forgotten_password – /users/forgotten-password – forgotten_password()
  • user.view – /users/{uid} – view()
  • user.confirm – /users/{uid}/confirm/{token} – confirm()
  • user.edit – /users/{uid}/edit – edit()
  • user.permissions – /users/{uid}/permissions – permissions()
  • user.delete – /users/{uid}/delete – delete()
wte.views.user.login(request)[source]¶

Handles the “/users/login” URL, checking the submitted username and password against the stored User and setting the necessary session variables if the login is successful.

wte.views.user.logout(request)[source]¶

Handles the “/users/logout” URL and deletes the current session, thus logging the user out

wte.views.user.permissions(request)[source]¶

Handles the “/users/{uid}/permissions” URL, providing the form and backend functionality for setting the Permission and PermissionGroup that the User belongs to.

wte.views.user.process_confirmation(request, user)[source]¶

The process_confirmation() function handles clearing the validation_token, generating a new password, and sending a welcome e-mail if the validation of the User was successful.

Parameters:
  • request (Request) – The request to use for configuration access
  • user (User) – The user to process
wte.views.user.process_forgotten_password(request, user)[source]¶

The process_forgotten_password() function handles generating a new password and sending the information e-mail. It can distinguish between User who have been validated and those that have not yet and will send the appropriate e-mail.

Parameters:
  • request (Request) – The request to use for URL generation
  • user (User) – The user to generate a new password for
wte.views.user.register(request)[source]¶

Handles the “/users/register” URL, displaying the registration form or if data is POSTed, creating a new user. Upon registration, a confirmation e-mail is sent to the given e-mail address.

wte.views.user.users(request)[source]¶

Handles the /users URL, displaying all users if the current User has the “admin.users.view” Permission.

wte.views.user.view(request)[source]¶

Handles the “/users/{uid}” URL, showing the user’s profile.
